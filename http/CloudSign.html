CloudSign¶

Имя ресурса: /CloudSign

HTTP метод: POST

Параметры запроса:

  • certificateThumbprint - Опциональный параметр. Строка. Отпечаток Контур.Сертификата, которым необходимо подписать документы. Если не задан, то будет использоваться текущий активный Контур.Сертификат пользователя, от имени которого делается вызов.

В запросе должен присутствовать HTTP-заголовок Authorization с необходимыми данными для авторизации.

В теле запроса должно содержаться сообщение CloudSignRequest, в формате протобуфер.

В теле ответа содержится AsyncMethodResult, в формате протобуфер.

Этот метод начинает асинхронную операцию подписания данных, переданных в запросе CloudSignRequest. Для завершения этой стадии подписания используйте метод CloudSignResult.

Возможные HTTP-коды возврата:

  • 200 (OK) - операция успешно завершена;

  • 401 (Unauthorized) - в запросе отсутствует HTTP-заголовок Authorization, или в этом заголовке содержатся некорректные авторизационные данные;

  • 405 (Method not allowed) - используется неподходящий HTTP-метод;

  • 500 (Internal server error) - при обработке запроса возникла непредвиденная ошибка.
