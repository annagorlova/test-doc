GenerateTorg12XmlForSeller¶

Предупреждение

Этот метод — устаревший. С 1 октября 2021 года он перестанет работать. Для генерации документов следует использовать метод GenerateTitleXml.

POST /GenerateTorg12XmlForSeller¶
Query Parameters

  • documentVersion – версия документа

  • disableValidation – отключение валидации полученного Xml документа по формату ФНС; параметр может отсутствовать

Request Headers

  • Authorization – необходимые данными для авторизации

Status Codes

  • 200 OK – операция успешно завершена

  • 400 Bad Request – данные в запросе имеют неверный формат или отсутствуют обязательные параметры

  • 401 Unauthorized – в запросе отсутствует HTTP-заголовок Authorization, или в этом заголовке содержатся некорректные авторизационные данные

  • 404 Not Found – при заполненном поле DiadocOrganizationInfo.BoxId в справочнике Диадока отсутствует организация, которой принадлежит указанный ящик

  • 405 Method Not Allowed – используется неподходящий HTTP-метод

  • 500 Internal Server Error – при обработке запроса возникла непредвиденная ошибка

Если documentVersion равен tovtorg_05_01_04:

  • в теле запроса должны содержаться данные для изготовления титула продавца для товарной накладной в XML-формате, в виде сериализованной структуры TovTorgSellerTitleInfo;

  • в теле ответа содержится XML-файл титула продавца для товарной накладной, построенный на основании данных из запроса. Файл изготавливается в соответствии с XSD-схемой (DP_TOVTORGPR_1_992_01_05_01_04.xsd), которая описывает рекомендованный ФНС формат для электронных товарных накладных.

Если documentVersion не указан или равен torg12_05_01_02:

  • в теле запроса должны содержаться данные для изготовления титула продавца для товарной накладной в XML-формате, в виде сериализованной структуры Torg12SellerTitleInfo;

  • в теле ответа содержится XML-файл титула продавца для товарной накладной, построенный на основании данных из запроса. Файл изготавливается в соответствии с XSD-схемой (DP_OTORG12_1_986_00_05_01_02.xsd), которая описывает рекомендованный ФНС формат для электронных товарных накладных.

Имя файла титула продавца для товарной накладной возвращается в стандартном HTTP-заголовке Content-Disposition;
